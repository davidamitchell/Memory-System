# Information

**Domain:** Knowledge representation and engineering

---

## Conceptual Definition

Information is data that has been assigned context and structure such that it is interpretable by an agent and reduces uncertainty about a state of affairs, without requiring verification of its truth.

**Necessary conditions:**
1. **Derived from data** — it originates in recorded, discrete observations or symbols.
2. **Contextualised** — a schema, ontology, or interpretive frame has been applied that assigns meaning to the symbols.
3. **Uncertainty-reducing** — it makes a distinction that matters to an agent: knowing it changes what the agent would expect or do.
4. **Interpretable** — a competent agent with the relevant schema can extract its content without additional context.

**Sufficient conditions:**  
Any contextualised, interpretable, uncertainty-reducing representation derived from data constitutes information.

---

## Operational Definition

Information is operationally identified when:

1. A schema or ontology exists that assigns meaning to the data's structure and values.
2. An agent with access to the schema can answer questions using it that they could not answer without it.
3. It can be encoded, transmitted, and decoded by parties sharing the same schema without loss of content.
4. It does not require external verification of truth to be used; it is accepted provisionally as a meaningful statement.

---

## Distinguished From

| Term | Distinction |
|------|-------------|
| **Data** | Data is uncontextualised raw symbols; information is data that has been contextualised and made interpretable. |
| **Knowledge** | Knowledge is information that has been verified as true and understood in relation to other knowledge; information does not require verification. |
| **Fact** | A fact is a verified true statement; information is a meaningful statement that may or may not have been verified. |
