# Data

**Domain:** Knowledge representation and engineering

---

## Conceptual Definition

Data is a collection of discrete, recorded symbols or measurements that have not yet been assigned context, interpretation, or relational structure sufficient to make them meaningful.

**Necessary conditions:**
1. **Recorded** — it has been captured and persists independently of the act of observation.
2. **Discrete** — it consists of separable units (symbols, measurements, values).
3. **Uninterpreted** — it lacks assigned meaning, context, or schema at the point of definition; it is the raw material prior to contextualisation.
4. **Finite and bounded** — a given data collection has defined membership at a point in time.

**Sufficient conditions:**  
Any collection of recorded, discrete symbols or measurements that lacks sufficient contextual assignment to be interpreted as information constitutes data.

---

## Operational Definition

Data is operationally identified when:

1. It can be stored and retrieved without a domain interpretation being required by the storage mechanism.
2. The same data can be interpreted differently by different agents applying different schemas (demonstrating it is pre-interpretive).
3. It serves as the input to an information-construction process that assigns schema, context, or meaning.

---

## Distinguished From

| Term | Distinction |
|------|-------------|
| **Information** | Information is data that has been contextualised and structured to be interpretable and to reduce uncertainty; data is the pre-contextualised raw material. |
| **Fact** | A fact is a verified true statement about the world; data is unverified and uninterpreted. |
| **Metadata** | Metadata is structured description about a resource; data is the raw content the metadata describes. |
