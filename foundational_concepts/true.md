# True

**Domain:** Knowledge representation and engineering

---

## Conceptual Definition

True is a property of a proposition that holds when and only when the proposition accurately corresponds to a state of affairs in the intended domain — or, in a formal system, when the proposition is satisfied by every intended model of that system.

**Necessary conditions:**
1. **Propositional** — it applies to propositions (statements that have a truth value), not to questions, commands, or non-assertoric expressions.
2. **Correspondence** — the proposition accurately represents the state of the world or domain it describes.
3. **Model-satisfaction** — in formal semantics: a proposition is true if and only if every interpretation that satisfies the axioms of the system also satisfies the proposition.
4. **Domain-relative** — the truth of a proposition is assessed against a specific domain and its intended models, not in the abstract.

**Sufficient conditions:**  
A proposition is true if and only if it corresponds to the actual state of affairs in the domain it describes, or equivalently, is satisfied by all intended models of that domain.

---

## Operational Definition

A proposition is operationally treated as true when:

1. It has been verified against evidence from the domain (empirical or formal proof).
2. No contradicting evidence or derivation exists within the knowledge base.
3. A reasoner finds it consistent with all stated axioms and facts.
4. It can be asserted without qualification in the knowledge base (i.e., added as an axiom or derived fact).

---

## Distinguished From

| Term | Distinction |
|------|-------------|
| **Fact** | A fact is a proposition that has been operationally verified and recorded as true; true is the property that makes something a fact. |
| **Valid** | Valid (in logic) applies to arguments or schema constraints; true applies to propositions or sentences. |
| **Asserted** | Asserted means recorded in the knowledge base; a statement can be asserted without being verified as true. |
